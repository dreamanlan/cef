注意事项：
1、master之外的2个分支主要是对cef的tests工程修改以方便执行附加的js，分别对应windows与macos平台
2、编译cef应选择某个稳定分支的源码编译，cef不推荐使用master编译
3、按cef的编译过程，这里的源码只是修改备份，另外开一个目录用来编译chromium与cef
4、按照https://bitbucket.org/chromiumembedded/cef/wiki/BranchesAndBuilding的说明选择分支并准备相应编译环境
5、按照https://bitbucket.org/chromiumembedded/cef/wiki/AutomatedBuildSetup.md的python脚本进行编译，但python脚本命令行需要加上--no-update，去掉--force-clean
6、master编译说明使用c:\code作为编译根目录，update.bat里会使用这个绝对路径，按需要调整成其它剩余空间足够的磁盘目录（特别注意不要使用github的源码目录），比如d:\cef_src_build之类，分支编译可以按照master文档的目录结构来组织
  root_dir -|
            |- automate
            |- chromium_git
            |- depot_tools
7、master编译文档https://bitbucket.org/chromiumembedded/cef/wiki/MasterBuildQuickStart可作为编译过程的参考，但分支按这个过程编译可能遇到gn命令报9009错误
8、本github工程的修改使用beyond compare将tests目录同步到cef下的tests目录即可，同步后需要重新编译，因此修改cef源码后用的编译脚本不能重新checkout覆盖cef目录，否则会导致修改丢失。如5所说
9、python脚本命令行看源码可能更清楚https://bitbucket.org/chromiumembedded/cef/raw/master/tools/automate/automate-git.py
10、cef使用的gn环境变量说明https://github.com/chromiumembedded/cef/blob/master/BUILD.gn，chromium的gn构建配置说明https://www.chromium.org/developers/gn-build-configuration/

windows下成功编译的步骤:
1、按照https://bitbucket.org/chromiumembedded/cef/wiki/BranchesAndBuilding的说明选择分支并准备相应编译环境
2、按照https://bitbucket.org/chromiumembedded/cef/wiki/MasterBuildQuickStart的1~5步建立编译目录结构，下载depot_tools与automate-git.py
3、新建chromium_git\update.bat，内容如下
  set GN_DEFINES=is_official_build=true chrome_pgo_phase=0
  set GN_ARGUMENTS=--ide=vs2022 --sln=cef --filters=//cef/*
  set GYP_MSVS_VERSION=2022
  set CEF_ARCHIVE_FORMAT=tar.bz2
  python ../automate/automate-git.py --download-dir=../chromium_git --branch=5845 --no-distrib --x64-build --build-target="cefclient cefsimple"
4、执行update.bat，将会更新depot_tools、chromium与cef源码并编译，时间会很长
5、对cef源码修改后，新建chromium_git\rebuild.bat，内容如下（不更新、不删除out目录直接重新编译）
  set GN_DEFINES=is_official_build=true chrome_pgo_phase=0
  set GN_ARGUMENTS=--ide=vs2022 --sln=cef --filters=//cef/*
  set GYP_MSVS_VERSION=2022
  set CEF_ARCHIVE_FORMAT=tar.bz2
  python ../automate/automate-git.py --download-dir=../chromium_git --branch=5845 --no-distrib --no-update --force-build --x64-build --build-target="cefclient cefsimple"
6、编译都需要很长时间（8小时以上）与大量磁盘空间（x64-build情形，out目录主要生成了x64的debug与release版本[目录结构是所有版本的，实际只有2个目录有生成内容]，约150GB，带git源码与编译工具整个目录220GB空间，剩余磁盘至少300GB空间可能是必要的）

实测运行时依赖的文件:
1、locales目录
2、snapshot_blob.bin与v8_context_snapshot.bin
3、icudtl.dat
4、resources.pak、chrome_100_percent.pak、chrome_200_percent.pak、headless_command_resources.pak
5、libcef.dll与chrome_elf.dll
6、支持WebGL需要d3dcompiler_47.dll、libEGL.dll、libGLESv2.dll、vk_swiftshader.dll、VkICD_mock_icd.dll、VkLayer_khronos_validation.dll、vulkan-1.dll
7、运行时库dbgcore.dll、dbghelp.dll、msdia140.dll、msvcp140.dll、vccorlib140.dll、vcruntime140.dll、vcruntime140_1.dll
8、保险一些的依赖文件包括locales目录、*.bin、icudtl.dat、*.pak与*.dll，也可以带上所有exe文件
